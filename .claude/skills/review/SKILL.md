# /review - 코드 리뷰

현재 변경사항 또는 지정된 파일을 리뷰합니다.

## 실행 단계

1. **변경사항 확인**
   ```bash
   git diff --staged
   git diff
   ```

2. **리뷰 수행**
   - 코드 품질 검토
   - 성능 이슈 확인
   - 보안 취약점 검토
   - Next.js/React 패턴 준수 확인

3. **리뷰 보고서 생성**

## 체크리스트

### Architecture
- [ ] Server/Client Component 올바른 구분
- [ ] 적절한 파일 구조
- [ ] 관심사 분리

### Code Quality
- [ ] 타입 안전성
- [ ] 에러 핸들링
- [ ] 코드 가독성

### Performance
- [ ] 불필요한 리렌더링
- [ ] 적절한 캐싱
- [ ] 번들 크기

### Security
- [ ] 입력 검증
- [ ] 인증/인가
- [ ] 민감 정보 노출

```markdown
## 코드 리뷰 결과

### 요약
[전체 평가]

### 개선 필요
- [High] 파일:라인 - 문제 설명
- [Medium] 파일:라인 - 문제 설명

### 잘된 점
- [칭찬 사항]

### 제안
- [개선 제안]
```
