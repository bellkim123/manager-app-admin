# /plan - 구현 계획 수립

기능 구현을 위한 상세 계획을 수립합니다.

## 실행 단계

1. **요구사항 파악**
   - 사용자 요청 분석
   - 명확하지 않은 부분 질문

2. **기존 코드 분석**
   - 관련 파일 탐색
   - 기존 패턴 파악
   - 영향 범위 확인

3. **아키텍처 설계**
   - 컴포넌트 구조
   - 데이터 흐름
   - API 설계 (필요시)

4. **구현 계획 작성**

## 계획 템플릿

```markdown
## 기능 개요
[구현할 기능 설명]

## 요구사항
- [ ] 요구사항 1
- [ ] 요구사항 2

## 영향 범위
### 새로 생성
- `path/to/new-file.tsx` - 설명

### 수정
- `path/to/existing-file.tsx` - 변경 내용

## 구현 단계
1. **단계 1**: 설명
   - 세부 작업 1
   - 세부 작업 2

2. **단계 2**: 설명
   - 세부 작업 1

## 기술적 고려사항
- Server vs Client Component 결정
- 데이터 페칭 전략
- 상태 관리 방식

## 의존성
- 추가 필요 패키지 (있는 경우)

## 예상 결과
[완료 후 예상되는 상태]
```

## 출력
구현 계획서를 마크다운 형식으로 출력합니다.
